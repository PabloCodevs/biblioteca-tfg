3.1. Interpolación de texto
La forma más básica de vinculación de datos es la interpolación de texto mediante la sintaxis
"Mustache" (doble llave):
La etiqueta "mustache" se reemplazará con el valor de la propiedad "msg" de la instancia del
componente correspondiente. También se actualizará cuando cambie la propiedad "msg".
3.2. HTML Puro (Raw HTML)
La sintaxis mustache interpreta los datos como texto plano, no como HTML. Para generar HTML
real, deberemos usar v-html:
Aquí nos encontramos con algo nuevo. El atributo v-html que se llama directiva. Las directivas
tienen el prefijo v- para indicar que son atributos especiales proporcionados por Vue y, como
habrás adivinado, aplican un comportamiento reactivo especial al DOM renderizado.
Básicamente, decimos "mantener el HTML interno de este elemento actualizado con la
propiedad msg de la instancia activa actual".
Atención!!
Representar dinámicamente HTML arbitrario en su sitio web puede ser muy peligroso, ya que
puede provocar fácilmente vulnerabilidades XSS. Usa v-html solo en contenido confiable y
nunca en contenido proporcionado por el usuario.
3.3. Vincular atributos (v-bind)
No podemos usar sintaxis mustache dentro de atributos HTML. En su lugar, utilice la directiva v-
bind:
<script setup lang="ts">
const msg: string = 'Hola mundo';
</script>
<template>
<h1>{{ msg }}</h1>
</template>
<script setup lang="ts">
const msg: string = '<h1>Hola mundo</h1>';
</script>
<template>
<div v-html="msg"</div>
</template>
La directiva v-bind indica a Vue que mantenga el atributo id del elemento sincronizado con la
propiedad dynamicId del componente. Si el valor enlazado es nulo o indefinido, el atributo se
eliminará del elemento renderizado.
3.3.1. Método abreviado
Dado que v-bind se usa con tanta frecuencia, cuenta con una sintaxis abreviada específica:
Los atributos que empiezan por : pueden parecer un poco diferentes del HTML normal, pero es
un carácter válido para nombres de atributo y todos los navegadores compatibles con Vue
pueden analizarlo correctamente. Además, no aparecen en el marcado final. La sintaxis
abreviada es opcional, pero es ampliamente usada.
3.3.2. Abreviatura del mismo nombre
Solo compatible con la versión 3.4 y posteriores
Si el atributo tiene el mismo nombre que la variable del valor de JavaScript/TypeScript que se va
a enlazar, la sintaxis se puede acortar aún más para omitir el valor del atributo:
<script setup lang="ts">
const redBackgroundClass: string = 'bgred';
</script>
<template>
<p v-bind:class="redBackgroundClass">Hola mundo</div>

7. Representación condicional (v-if, v-else, v-else-if)
Vue usa las directivas v-if, v-else y v-else-if para trabajar con condicionales. En el siguiente
ejemplo vamos a crear un programa con 3 botones (A, B y C) de manera que se muestre por
pantalla si hemos pulsado el botón A, B o C:
<script setup lang="ts">
import { ref } from 'vue';
const paint = ref(false);
</script>
<template>
<button @click="paint = !paint"> Pintar </button>
<p :class="[
paint ? 'bg-red' : 'bg-blue', 'text-white']"> Hola mundo!
</p>
</template>
<style scoped>
.bg-red {
background-color: red;
}
.bg-blue {
background-color: blue;
}
.text-white {
color: white;
}
</style>
<script setup lang="ts">
import { ref } from 'vue';
const type = ref('A');
</script>
<template>
<button @click="type = 'A'"> A </button>
<button @click="type = 'B'"> B </button>
<button @click="type = 'C'"> C </button>
<p v-if="type === 'A'">
Se ha pulsado el botón A
</p>
<p v-else-if="type === 'B'">
Se ha pulsado el botón B
</p>
<p v-else>
Se ha pulsado el botón C
</p>
</template>
Obviamente, no es la mejor forma de resolver el problema, pero es válido para explicar el uso de
la representación de condicionales en vue.js.
Nota
En vue.js también existe la directiva v-show que permite mostrar y ocultar elementos. La
diferencia es que el elemento siempre existirá y lo ocultará mediante css.

05-Pinia🍍
5.1. ¿Qué es Pinia?
Según wikipedia: Pinia es una biblioteca de tiendas y un marco de gestión de estado para
Vue.js. Diseñado principalmente para crear aplicaciones web frontales, utiliza sintaxis declarativa
y ofrece su propia API de administración de estados.
Hasta ahora, cada vez que queríamos guardar una información, lo que hemos hecho es crear
una variable dentro de nuestro componente y; si hemos necesitado que la información sea
usada por otro componentes, hemos hecho la comunicación entre ellos (ya sea por eventos, v-
model...).
Va a haber ocasiones en las que vamos a querer almacenar cierta información que vamos a
querer que sea compartida por muchos (o casi todos) los componentes, como por ejemplo
guardar la información de un usuario (autenticación de usuario) y es ahí donde Pinia es muy útil.
Antes para esto se usaba VuEx, que actualmente ha evolucionado a Pinia.
5.2. Definición de una tienda

Esta estructura que tenemos aquí es válida, pero se va a utilizar sobre todo cuando estamos
